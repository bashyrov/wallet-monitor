Screener controls
Globally hide tokens, disable exchanges, toggle maintenance — applied to every user within 15s
Hidden symbols0
Drop these tokens from every funding / arb feed. Uppercase, without the quote suffix. E.g. STOCK_CSCO, MEMES.
Exchange availability
Turn an exchange off to stop fetching funding rates from it (and to drop its rows from the screener + arb). Useful when an exchange misbehaves or is under maintenance itself.
Trading availability
Turn an exchange off to stop users from opening / closing positions through our service on it. The exchange still shows up on the screener, funding and portfolio — this only blocks the trade endpoints. Useful when an adapter has a known bug or we're pausing a venue pending an audit.
Portfolio controls
Disable specific providers for wallet connections — existing wallets keep working, new connections are rejected.
CEX exchanges
Exchanges available as wallet sources (Binance, OKX, etc.). Turning one off hides it from the Add Wallet modal and rejects new API-key submissions.
Perpetual DEXes
Hyperliquid, Aster, Lighter, Ethereal, Paradex — as wallet sources. Screener-side availability is controlled in the Screener tab.
Chains
EVM + non-EVM networks. Disabling a chain hides it from new wallet creation.
Maintenance
Kill switches for the whole site or individual sections. Changes apply within ~15s. /api/admin/* always stays reachable.
Used in the user-facing lockout pages: "Tech work ends at 13:00 (Europe/Warsaw)". Examples: Europe/Warsaw · Europe/Kyiv · UTC.
Full-site maintenance
Hard kill switch — every page returns a 503 maintenance screen. /api/admin/* stays reachable. File flag /tmp/avalant_maintenance still works as a fallback.
—
Screener section
Blocks /screener, /arb, /watchlist, and /api/screener/*. Portfolio and admin keep running.
—
Portfolio section
Blocks /app, /archive, /profile, /avashare, and wallet/alert/trade APIs. Pricing + checkout stay open so users can renew.
—
Plans
Pricing tiers, limits and feature lists. Free plan can't be deleted.
Promo codes
One code per checkout. Discount % rounds to 2 decimals at the cart level.
Popups
Promotional popups for logged-in users. Frequency=once means one show per user; every-N-min re-shows after the dismiss timer elapses.
Billing periods
Commitment lengths shown on /pricing. Each row defines a months count + discount %. Pricing is base_monthly × months × (1 − discount/100), rounded to 2 decimals.
Telegram broadcast
Push a message via the auth bot to every linked user, or to one specific user. HTML formatting supported (<b>, <i>, <a href>, <code>).

Site banner

A green strip above the navbar on every page. Live for everyone — auth and anon. Switch on Marquee for endless scroll, off for static centered text. Cap 500 chars.

Expiry-reminder schedule

Send a one-off message

Sent via the auth bot. HTML mode — close every tag, escape </> in content.
Avashare payouts
Referral commission payout requests. Confirm payment manually (TRC20) and mark each as paid; reject with a reason if needed.
