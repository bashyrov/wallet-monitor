Security

Last reviewed 2026-05-17.

What we never store

  • Wallet private keys. Avalant never asks for or holds your on-chain private keys. Portfolio addresses are read-only — we look up balances and transactions via public RPCs and exchange APIs.
  • Exchange withdrawal permissions. We only ever use API keys with read + trade permissions. If your key grants withdrawal, that's on the venue side — we don't call those endpoints.
  • Plaintext passwords. Account passwords are bcrypt-hashed (work factor tuned per process budget). Reset tokens are SHA-256 hashed and TTL-capped at 1 hour.

How API keys are protected

  • Fernet encryption at rest. Every exchange API key/secret is encrypted with a per-deployment ENCRYPTION_KEY (PBKDF2-SHA256, 260k iterations) before the DB row is committed. The key never appears in plaintext on disk.
  • Key rotation. Rotating the encryption key is a one-command operation (scripts/rotate_encryption_key.py) that re-encrypts every row safely — partial failures are idempotent.
  • Same protection for 2FA TOTP secrets. Stored Fernet-encrypted in users.totp_secret_enc; tear-down on disable.

Auth + sessions

  • JWT bearer auth (HS256), 30-day rolling lifetime. Revocation list backed by Redis (jti blacklist) so a leaked token can be killed immediately.
  • HttpOnly + Secure session cookie in production. Cross-origin XHR uses Authorization header; cookies never carry the trade-API token.
  • Per-account login throttle. 5 failed attempts → lock; the counter is stored on the user row and clears on a successful login.
  • Admin requires TOTP. Admin sessions are gated on a second factor; failed TOTP triggers a TG alert to the admin channel.
  • Admin promotion is SQL-only. There is no API path, no env var, no registration flow that grants admin. The only way is a direct UPDATE on the host.

Network + headers

  • HSTS (1 year, includeSubDomains), X-Frame-Options DENY, X-Content-Type-Options nosniff, restricted Permissions-Policy.
  • CSP with frame-ancestors 'none', narrowly-scoped connect-src, no third-party script CDNs except telegram.org (Login Widget) and self-hosted vendor bundles.
  • Cloudflare in front of origin for rate limiting and TLS termination. nginx adds per-zone request limits for /api/auth/*, /api/portfolio/balance, and generic /api/*.

Reporting a vulnerability

If you find a bug that affects account safety, key handling, or anything that looks like privilege escalation — please tell us before disclosing publicly. We respond quickly.

Disclosure contact

Email security@avalant.xyz with a short repro. Please don't post details in public channels until we've had a chance to fix and ship.

What's not security-critical

We don't run a paid bug bounty yet, but we do credit responsible disclosure on a future hall-of-fame page. Reports about missing CSRF tokens on the JSON API (we use Bearer auth, not cookie form-post) or about 'unsafe-inline' in our CSP (intentional until inline-script removal lands) are known and not eligible.
